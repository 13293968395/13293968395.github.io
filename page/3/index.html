51.N皇后

Posted on 2021-10-09 | In 数据结构与算法

思路

n 皇后问题 研究的是如何将 n 个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。

给你一个整数 n ，返回所有不同的 n 皇后问题 的解决方案。

每一种解法包含一个不同的 n 皇后问题 的棋子放置方案，该方案中 'Q' 和 '.' 分别代表了皇后和空位。

国际象棋中的皇后，她横、竖、斜都可走一或七步，可进可退。

所以任意皇后之间不能相互攻击就是她们不能位于同一对角线或者同一直线上

基于集合的回溯

皇后不能位于一直线上，所以在每一行上只有一个皇后，我们可以遍历每一行。

那么那么，只需要记录某一列是否可以放置皇后

  • col[i] 表示第i列是否可以放置


在上图中，假如这两条直线就是对角线，x是列，y是行。

那么那么，y=x+b 与y=-x+b 就有一个恒等式, y-x=b与y+x=b.

也就是一条对角线上的，行+列=相同且唯一的常数。

这个常数就可以表示某一条对角线

  • dg[x+y] 表示x+y的对角线上是否可以放置皇后
  • dg[y-x+n] 与上式相同，其中n是某一常数，目的使y-x不为负。

贪心思想

非凸函数 ：任意画一条线，不能保证所有的点都在曲线的上方或曲面的上方

鞍点： 没有梯度，无法更新

梯度下降：

指数加权均值使得cost曲线更加平滑

随机梯度下降 SGD

! 梯度

  • 梯度下降，使用所有的平均损失对权重求导

    • 每个数据之间是无关系的，数据之间可以并行训练，时间复杂度低，但不够优化

  • 随机选择一个数据, 将其的损失对权重求导，有利于跨越鞍点，

    • 数据之间有关系，无法并行，时间复杂度高，但效果高。

所以使用一个折中的方法,将若干个数据组成batch，每次用一个batch进行训练，求梯度
