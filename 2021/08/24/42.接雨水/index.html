42.接雨水

Posted on 2021-08-24 | In 数据结构与算法

https://leetcode-cn.com/problems/trapping-rain-water/

1. 暴力法

当左右两高柱的高度大于中间高柱时（至少需要3个高柱）,便会形成积水。由于木桶效应,积水的高度由两侧高柱的最小高度决定。
那么就可以遍历每一个高柱，并求当前遍历到的高柱的左右两侧的最高柱形.求出面积并相加

2. 动态规划

以上图为例

通过暴力法发现: 我们需要求每一个柱子的左右最高柱, 其中有很多重复计算, 比如第7个高柱之前所有的高柱的maxRight均为3,
那我们可不可, 通过保存每个柱子的左右最高柱, 从而避免重复计算.maxRight, maxLeft分别保存最大左右柱的下标

公式如下

  • maxLeft[i] = max(maxLeft[i-1], height[i]) (0 < i < len)
  • maxRight[i] = max(maxRight[i+1], height[i]) (0<= i < len-1)

正向遍历得到maxLeft, 反向遍历得到maxRight

3. 单调栈

遍历此高度数组

  • i=0, height[i]=0, 当前无法判断是否有积水
  • i=1, height[i]=1, 比上一个高柱高, 而且上一个高柱为最左端, 则说明上一个高柱无法积水
    。i=0 的高柱就不需要再判断
  • i=2, height[i]=0, 比上个高柱低, 无法判断当前高柱与上一高柱的积水

  • i=3, height[i]=2, 比上个高柱高,以i=2为中间的高柱,存在均比中间高的高柱, 则存在积水
    。计算以i=2的高柱对应的积水量, width = 3-1, height =min(height[3], height[1]) - height[2]
    然后就可以将i=2高柱剔除出考虑范围.
  • 据上: 遇到比上一高柱高的高柱时, 就必然会有积水, 除非上一高柱为左边界, 这么说的话, 就可以使用栈作为辅助的数据结构, 遍历数组, 遇到比栈顶大的高柱时,则就计算其对应的积水量, 并将其弹出.若弹出后栈为空, 则说明其左边无比它更高的高柱, 退出循环即可

4.双指针

动态规划中，我们为了避免重复的计算使用了两个数组maxRight maxLeft两个数组保存每个高柱的左右最高柱,
那我们可不可以用常数个变量保存

还记的暴力法吗, 暴力法中对每一个高柱都求其maxLeft maxRight, 但由于木桶效应, 其积水量只有其中较小的决定
那我们就可以使用两个指针left right分别在数组两端向中间遍历,两个变量maxLeft maxRight记录当前为止的最大高度,

  • 当height[left] < height[right]时, 此时由于maxLeft=max(maxLeft, height[left]) 与 maxRight=max(maxRight, height[right]), 那么一定存在maxLeft < maxRight
    , 所以当前高柱对应的积水量由maxLeft决定, res = maxLeft - height[left]
  • 当height[left] >= height[right]时, 同上,当前高柱对应积水由maxRight决定,res = maxRight - height[right]
